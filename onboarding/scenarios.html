Run / 01

Fine-tune ViT-small on StateFarm with LoRA

Four commands. No code changes. No YAML edits.

Step 1 Confirm pieces exist

Every plug point is registered by string. Check the backbone, the dataset, and the fine-tune strategy are in the three registries this run touches.

python -m dms.registry list backbones   # vit_small — yes
python -m dms.registry list datasets    # statefarm — yes
python -m dms.registry list fine_tune   # lora — yes
Step 2 Run

Hydra composes the full config from YAML groups. Flat overrides on the CLI tune batch size, learning rate, epochs.

python scripts/training/train.py \
    model=vit_small \
    dataset=statefarm \
    trainer=finetune_lora \
    trainer.epochs=20 \
    trainer.optimizer.lr=3e-4 \
    trainer.batch_size=16
Step 3 Inspect outputs

Outputs land at experiments/runs/vit_small/. Three things to know:

  • best.pt — the checkpoint at the best validation epoch.
  • metrics.json — per-epoch losses and accuracies.
  • .hydra/ — the fully-composed config. Use this to reproduce the run months later.
Step 4 Evaluate

Same Hydra shape, different script. Point at the checkpoint; the dataset + model configs determine the eval loader and the model architecture.

python scripts/evaluation/evaluate.py \
    model=vit_small \
    dataset=statefarm \
    checkpoint_path=experiments/runs/vit_small/best.pt
checkpoint_path is a TOP-LEVEL override. Not a nested model.* field. A common stumble on the first evaluation run.

Extend / 02

Add a new metric: precision_at_recall

Six steps. The pattern works for every extension point.

1
Create the file

One class, one file. Path mirrors the existing metrics:

src/dms/evaluation/metrics/precision_at_recall.py
2
Subclass BaseMetric with required class attrs
from dms.evaluation.metrics.base import BaseMetric, MetricResult
from dms.registry import metric_registry

@metric_registry.register("precision_at_recall")
class PrecisionAtRecall(BaseMetric):
    name = "precision_at_recall"
    requires = ["predicted_labels", "true_labels"]
    higher_is_better = True
Missing any of name, requires, higher_is_better raises TypeError at class-definition time. CLAUDE.md §8 #29.
3
Implement the two methods

compute(predictions) -> MetricResult and format(result) -> dict. The bundle carries predicted_labels, true_labels, probabilities, and more.

4
Decorate — already shown in step 2

@metric_registry.register("precision_at_recall") binds the string name to the class. YAML configs refer to the string.

5
Import in __init__.py

Registration fires on module import. Add the line to src/dms/evaluation/metrics/__init__.py so the class loads when the package loads.

6
Reference in a YAML + run
# configs/evaluation/my_eval.yaml
metrics:
  - accuracy
  - precision_at_recall
python scripts/evaluation/evaluate.py evaluation=my_eval ...
Same six steps for adapters, backbones, temporal modules, losses, schedulers, callbacks, visualizers, analyzers, compressors. Subclass base → register → import → reference in YAML.

Gotchas / 03

Top 5 traps. One of three.

The subset of CLAUDE.md §8 you will hit in your first week.

Gotcha 1 Hydra augmentation has double nesting
CLAUDE.md §8 #18
Symptom
TypeError: AugmentationConfig.__init__() got an unexpected keyword argument 'augmentation'
Cause

cfg.augmentation from Hydra returns {"augmentation": {...}}, not {...}.

Fix
aug_dict = OmegaConf.to_container(cfg.augmentation, resolve=True)
aug_dict = aug_dict["augmentation"]      # unwrap once
aug_cfg = AugmentationConfig(**aug_dict)
Gotcha 2 ManifestReader.read() returns a list, not a DataFrame
CLAUDE.md §8 #1
Symptom
AttributeError: 'list' object has no attribute 'loc'
Cause

.read() returns list[FrameSample] | list[VideoClipSample] — Pydantic samples, one per manifest row.

Fix

Use ManifestReader(...).read_as_dataframe() for a pd.DataFrame.

Gotcha 3 build_composed_from_pretrained returns a 3-tuple
CLAUDE.md §8 #2, #31
Symptom
AttributeError later when trying model.forward(...) because model is actually a tuple.
Cause

The factory returns (ComposedModel, PreprocessingConfig, id2label).

Fix
model, preprocessing, id2label = build_composed_from_pretrained(
    source, model_id, num_classes
)
Extra

Fine-tuning callers must pass preserve_pretrained_head=False. Without it, the factory transplants the pretrained classifier head and num_classes must match the pretrained head's out_features.

Gotcha 4 Windows DataLoader needs num_workers=0 AND prefetch_factor=null
CLAUDE.md §8 #21
Symptom
Training process hangs, or a pickling error on the first batch.
Cause

TransformPipeline uses lambda transforms that cannot be pickled for spawn-based multiprocessing.

Fix
python scripts/training/train.py \
    data.num_workers=0 \
    data.prefetch_factor=null \
    ...

Gotcha 5 EvalReport.get_metric(name) returns None when missing
CLAUDE.md §8 #26
Symptom
AttributeError: 'NoneType' object has no attribute 'value' downstream.
Cause

.get_metric() returns None if that metric was not in the evaluation request. It does NOT raise.

Fix
metric = report.get_metric("accuracy")
if metric is None:
    raise RuntimeError(
        "'accuracy' was not requested in the metrics list"
    )
