Rules / 03

Enforced in review

Three rules. Both CLAUDE.md and CONTRIBUTING.md cite them. Breaking them is the fastest way to get a PR bounced.

Rule 1 Never swallow errors or degrade gracefully
✘ Bad
try:
    weights = load(path)
except Exception:
    weights = default_weights()  # silent fallback
✔ Good
weights = load(path)
# let FileNotFoundError propagate
Rationale. If something fails, the caller must know something went wrong. No silent fallbacks, no except: pass, no returning defaults on failure.
Rule 2 Always document edge case handling
✘ Bad
def pick(xs):
    # no comment, no raise
    return xs[0] if xs else None
✔ Good
def pick(xs):
    if not xs:
        raise ValueError("xs empty")
    return xs[0]
Rationale. Every function states how edge cases are handled — via docstring, comment, or explicit raise. If a case is intentionally unsupported: raise NotImplementedError(...) or ValueError(...). Never silently produce wrong results.
Rule 3 Never delete raw data
✘ Bad
# wipes the source dataset
shutil.rmtree("datasets/raw/yawdd")
extract(new_frames, to="datasets/raw/yawdd")
✔ Good
# read from datasets/, write new dir
extract(
  src="datasets/raw/yawdd",
  dst="data/processed/yawdd_faces",
)
Rationale. datasets/ is append-only. Processing scripts write to data/processed/<new_name>/. If a script could destroy data (shutil.rmtree, os.remove on a data dir), it must refuse and ask the user. The two data directories are intentional: datasets/ = immutable source, data/ = scratchpad.
These three show up in review comments more than any stylistic nit. Internalise them before writing your first PR.
