Pipeline

Five stages, one shape

Every DMS experiment walks this path. Each arrow is a boundary; each boundary is pluggable.

Stage roster

Stage Role Home directory
Adapter raw dataset → manifest.csv src/dms/data/adapters/
DataModule manifest.csv → train / val / test loaders src/dms/data/datamodule.py
ComposedModel backbone → optional temporal → classification head src/dms/models/
Trainer one loop + FineTuneStrategy → checkpoint under experiments/runs/ src/dms/training/trainer.py
EvaluationRunner PredictionBundle → EvalReport (metrics, analyzers, visualizers) src/dms/evaluation/runner.py
Why one shape. Every extension point — adapter, backbone, metric — plugs into exactly one of these five stages. That is why there are 13 registries, not 100 utility functions.

Adapter / 01

Raw dataset → manifest.csv

The first boundary. Every dataset enters through an adapter and leaves as a CSV with the same schema.

What it does

Normalizes any raw dataset into one manifest.csv with a fixed schema (sample_id, frame_path, label, label_id, split, subject_id, session_id, ...). After this boundary, the pipeline does not know which dataset it is.

Contract

3 abstract methods — download(), convert(), get_label_map() — plus 4 class attrs: name, supported_tasks, supported_modalities, supported_cameras. Enforced via __init_subclass__. Missing attr = TypeError at class-definition time.

Minimal adapter
from dms.data.adapters.base import BaseDatasetAdapter
from dms.registry import dataset_registry

@dataset_registry.register("my_dataset")
class MyDataset(BaseDatasetAdapter):
    name = "my_dataset"
    supported_tasks = ("classification",)
    supported_modalities = ("frame",)
    supported_cameras = ("front",)
    def download(self): ...
    def convert(self): ...
    def get_label_map(self): ...

DataModule / 02

manifest.csv → PyTorch DataLoaders

Applies preprocessing + augmentation, emits three loaders.

Construction
dm = DataModule(
    config=DataConfig(...),
    preprocessing_config=PreprocessingConfig(...),
    augmentation_config=AugmentationConfig(...),
)
dm.setup()
train_loader = dm.train_loader()   # NOT train_dataloader()
val_loader   = dm.val_loader()
test_loader  = dm.test_loader()
Lightning muscle memory. Users will type train_dataloader() out of habit. That method does not exist here. Call train_loader(), val_loader(), test_loader(), and .setup() before any of them. (CLAUDE.md §8 #6)
Gotchas at this boundary
§8 #1
ManifestReader.read() returns list[FrameSample] | list[VideoClipSample], not a DataFrame. Use .read_as_dataframe() to get a pd.DataFrame.
§8 #18
Hydra augmentation config has double nesting. cfg.augmentation returns {"augmentation": {...}}. Must unwrap: aug_dict = aug_dict["augmentation"] before passing to AugmentationConfig().
§8 #21
Windows: DataLoader num_workers must be 0. Lambda-based transforms from TransformPipeline can't be pickled for multiprocessing spawn. Set num_workers=0 and prefetch_factor=None.
§8 #23
AugmentationConfig validates speed_jitter_range[0] < [1] even when speed jitter is disabled. Don't use [1.0, 1.0] as a "disabled" sentinel — use valid defaults like [0.8, 1.2].

ComposedModel / 03

backbone → [temporal] → head

Three pluggable parts. One constructor for 8 backbones, 4 temporal aggregators, multiple heads.

Backbone · features
8 registered
MobileNetV3 · ResNet · ViT · EfficientNet · ConvNeXt · VideoMAE · EfficientFormerV2 · YOLO-cls
Temporal · aggregator (optional)
4 registered
mean_pool · lstm · tcn · transformer
Head · classifier
Linear
MultiTaskModel branches by num_classes type
Build from config
from dms.models.factory import build_model

model = build_model(cfg, num_classes=10)
# → ComposedModel

model = build_model(cfg, num_classes={"drowsy": 2,
                                      "distract": 5})
# → MultiTaskModel
Load pretrained
model, preprocessing, id2label = \
    build_composed_from_pretrained(
        source, model_id, num_classes
    )
# 3-tuple. Binding to a single variable
# is the bug that bites everyone once.
Silent architecture switch. The type of num_classes silently selects a different architecture. int → ComposedModel. dict → MultiTaskModel. (CLAUDE.md §8 #30)
Pretrained head is preserved by default (fixed 2026-04-20). build_composed_from_pretrained transplants the classifier's Linear.weight/bias into the fresh LinearHead. Fine-tuning callers must pass preserve_pretrained_head=False; num_classes must equal the pretrained head's out_features or it raises. (CLAUDE.md §8 #2, #31)
Gotchas at this boundary
§8 #19
LoRA targets must match the backbone architecture. Transformer backbones: qkv, proj, fc1, fc2. MobileNetV3 (CNN): conv_reduce, conv_expand, conv_pw, conv_pwl. Using wrong targets causes ValueError: Target modules not found.
§8 #20
PEFT task_type must be None for non-HuggingFace models. Using TaskType.FEATURE_EXTRACTION wraps the model in PeftModelForFeatureExtraction which passes input_ids kwargs — breaks custom backbones like TimmBackbone.

Trainer / 04

One loop, many strategies

FineTuneStrategy plugs into a single Trainer class. That is how LoRA, BitFit, progressive unfreezing share one training loop.

Construction
from dms.training.trainer import Trainer

trainer = Trainer(model, train_loader, val_loader,
                  optimizer, scheduler, loss, callbacks, ...)
result = trainer.fit()   # → TrainResult; checkpoint at experiments/runs/<model.name>/
Fine-tune strategies (6)
frozen_backbone full progressive lora bitfit head_to_toe
Strategy lifecycle
optimizer = strategy.apply(
    model, optimizer_cls, optimizer_kwargs
)
# 3 args → returns an Optimizer.
# The strategy mutates requires_grad AND
# builds param groups.

Three args, not one. Very common mistake to pass only the model. (CLAUDE.md §8 #15)

LoRA config hand-off
# FineTuneConfig.lora from Hydra is a plain
# dict, not a LoRAConfig dataclass.
lora_dict.pop("merge_on_export", None)
cfg_lora = LoRAConfig(**lora_dict)
# Strip extra keys (e.g. merge_on_export)
# that are not in LoRAConfig.

CLAUDE.md §8 #22.

Import site matters. FineTuneConfig and LoRAConfig live in dms.schemas.config, not in dms.training.fine_tune. (CLAUDE.md §8 #14)

Evaluation / 05

PredictionBundle → EvalReport

Classes, not functions. Metrics compute numbers, analyzers produce reports, visualizers produce images.

PredictionBundle shape
PredictionBundle(
    logits, probabilities,
    predicted_labels, true_labels,
    sample_ids, label_map,
    # optional:
    features, temporal_logits, metadata,
)

CLAUDE.md §8 #12.

Usage
runner = EvaluationRunner(
    metrics=["accuracy", "f1_macro",
             "confusion_matrix"],
    visualizers=[...], analyzers=[...],
)
report = runner.run(predictions)   # → EvalReport
metric = report.get_metric("accuracy")
if metric is None:
    raise RuntimeError(
        "accuracy was not requested")
EvalReport.get_metric(name) returns None when the metric does not exist — it does NOT raise. Always check. (CLAUDE.md §8 #26)
No shortcut attrs. EvalReport has metrics: list[MetricResult]. There are no .accuracy / .f1_macro attributes. (CLAUDE.md §8 #11)
Metric contract
@metric_registry.register("my_metric")
class MyMetric(BaseMetric):
    name = "my_metric"
    requires = ["predicted_labels", "true_labels"]
    higher_is_better = True
    def compute(self, predictions): ...
    def format(self, result): ...

All three of name, requires, higher_is_better are enforced via __init_subclass__ on BaseMetric. Missing attr → TypeError at class-definition time. Same for BaseVisualizer, BaseAnalyzer, BaseDatasetAdapter. (CLAUDE.md §8 #29)

Miscellaneous gotchas
§8 #3
Aggregators from get_aggregator() are plain functions called as aggregator(logits, labels). They are NOT objects with an .aggregate() method.
§8 #4
predict_frames() / predict_clips() return softmax probabilities (not raw logits, despite the variable name in pipeline.py).
§8 #28
ManifestReader.read(validate=True) raises ManifestValidationError (not a generic ValueError) when rows don't conform. The error collects all row-level issues.
