Class Distribution

Drive&Act 8 DMS classes — significant imbalance addressed with sqrt-inverse-freq weighting

safe_driving dominates at 3,707 clips while looking_around has only 107 (35x imbalance). We use sqrt-inverse-frequency class weighting in the loss function to prevent the model from ignoring minority classes.

Our Preprocessing Pipeline

NIR-specific preprocessing before temporal shift module classification

NIR frames are extremely dark (88% of pixels below 50/255). CLAHE redistributes contrast, then frames are normalized with ImageNet statistics at full 640×480 resolution.

Implications for Model Design

  • NPU budget of 1.5–4 TOPs rules out most video models at full resolution — TSM with a lightweight backbone (MobileNetV3) remains viable
  • INT8 quantization is mandatory — all candidate models must be quantization-friendly (no significant accuracy drop at INT8)
  • Image-based classifiers are the pragmatic default for edge; video-based models shift to the centralized path unless edge TOPs increase
  • Complex events (multi-step distraction sequences) are candidates for centralized inference via streamed or batched frames

Roadmap

Timeline & Milestones

Phased convergence — parallel workstreams meeting at defined checkpoints

Phase 0 — Research & Baseline DONE

Completed

Open-source dataset evaluation, pretrained model replication (5 models, 3 datasets matched/exceeded), TSM temporal modeling on Drive&Act, pipeline architecture designed.

Phase 1 — Data & Assessment IN PROGRESS

Current phase

Data: Consolidate open-source + Howen data, set up storage, implement augmentation pipeline.
Models: Evaluate pretrained models on Howen subset, assess HW compatibility.
HW: Complete device specifications and NPU constraint documentation.

Phase 2 — Annotation & Fine-tuning NEXT

After consolidated dataset

Data: Model-assisted annotation → human validation loop, quality assurance.
Models: Fine-tune edge + centralized models on consolidated data, INT8 quantization experiments.
HW: On-device inference testing.

Phase 3 — Benchmarking & Integration NEXT

Production readiness

End-to-end benchmarks (accuracy, latency, size), edge deployment validation, centralized pipeline for complex events, production monitoring setup.

Key Dependencies

  • Consolidated dataset gates fine-tuning — both data sources must be merged before Phase 2 begins
  • HW specs constrain model selection — compatibility matrix needed before fine-tuning targets are locked
  • Annotation quality gates production training — human validation must hit target coverage before full fine-tuning
