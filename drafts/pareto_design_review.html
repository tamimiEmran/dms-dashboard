Pareto chart redesign — 3 options

Three alternative visualizations of the same ensemble-growth data, framed for a manager review. Each option re-imagines the existing Pareto chart so the audience can see how each newly added model improves overall performance and improves each use case specifically. Pick the one whose narrative reads most cleanly at a glance.

Source data · PARETO_DATA from hikvision_leaderboard_benchmark_summary.html · Hikvision OEM mean = 73.62%
Option 1

Per-class small multiples

Five panels — one per use case — each comparing the ensemble's accuracy across n = 1..5 against the Hikvision OEM baseline for that specific class. Shared 0–100% y-axis so heights stay comparable across panels. A badge under each title states whether the ensemble eventually beats OEM, and by how much it trails when it doesn't.

Ensemble accuracy (n=1..5) Hikvision OEM baseline Shaded fill: green when ensemble ≥ OEM at n=5, red when below
Option 2

Mean trajectory with per-step "what improved" annotations

A single chart showing the ensemble's mean accuracy versus the Hikvision OEM baseline. At every marker, an annotation calls out which model was added at that step and which use case it lifted (with the per-class point-delta). The reader can instantly see which addition delivered which capability.

Ensemble mean accuracy Hikvision OEM baseline (73.62%)
Option 3

Mean vs. worst-class accuracy — "the floor rises"

Same mean line, but paired with a second line showing the worst-performing use case at each step. The gap between mean and floor narrows as the ensemble grows — the strongest evidence that adding models doesn't just lift the average, it eliminates catastrophic per-class failures.
